Importa Materie Prime da CSV/Excel

📋 Formato File Richiesto

Il file CSV o Excel deve contenere le seguenti colonne (con intestazione):

color_name,color_ref,color_id,lego_size,size_code,picture_url,unique_id

Tutti i 7 campi sono OBBLIGATORI:

  • color_name → Nome colore LEGO (es. Light Nougat)
  • color_ref → Codice HEX colore (es. #F6D7B3)
  • color_id → ID colore LEGO (es. 78)
  • lego_size → Descrizione size (es. Brick 1*1)
  • size_code → Codice size LEGO (es. 3005)
  • picture_url → URL immagine LEGO
  • unique_id → ID univoco pezzo (es. 300578)

Esempio:

color_name,color_ref,color_id,lego_size,size_code,picture_url,unique_id Light Nougat,#F6D7B3,78,Brick 1*1,3005,https://www.bricklink.com/PL/3005.jpg,300578 Light Nougat,#F6D7B3,78,Brick 1*2,3004,https://www.bricklink.com/PL/3004.jpg,300478

Note:

  • Usa virgola (,) come separatore
  • Prima riga: intestazione colonne
  • Tutti i 7 campi devono essere compilati
  • Color HEX deve iniziare con # (es. #F6D7B3)
  • Encoding: UTF-8

ℹ️ Come Funziona l'Import

Il sistema importa solo materiali con unique_id nuovo:

  • ✅ Unique ID NON presente → Materiale viene importato
  • ⚠️ Unique ID già esistente → Riga viene saltata (dati esistenti non modificati)

Esempio: Se hai 300578 nel database e lo reimporti, viene saltato. Questo protegge i dati esistenti da sovrascritture accidentali.

Formati supportati: CSV, XLSX, XLS
